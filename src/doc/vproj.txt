1. INTRO                                             *vproj-intro*

vproj is a project manager sidebar for Vim. Browse files, switch buffers,
and manage project structure from a vertical pane.

Press |Tab| to toggle the pane (temporary mode), |Shift-Tab| for permanent
mode, or use |:VprojToggle|. In temporary mode, the pane auto-closes when
you open a file, press ESC, or press Tab again. In permanent mode, the
pane stays open until you press Shift-Tab again or run |:VprojClose|.

Installation: clone the repository and run install.sh, or create symlinks
manually to ~/.vim/pack/bundle/start/vproj/.

==============================================================================
2. COMMANDS                                         *vproj-commands*

:VprojToggle                                        *:VprojToggle*
    Toggle the project pane open (temporary mode) or closed.
    If the pane is open in permanent mode, switches to temporary.
    See |vproj-pane-modes|.

:VprojOpen                                          *:VprojOpen*
    Open the project pane.

:VprojClose                                         *:VprojClose*
    Close the project pane.

:VprojRefresh                                       *:VprojRefresh*
    Refresh the pane contents (re-read directory).

:VprojDiag                                          *:VprojDiag*
    Diagnostic tool. Tests split creation and reports window
    dimensions, cmdheight, and winmin settings. Useful for
    debugging E36 "Not enough room" errors.

3. MODES                                             *vproj-modes*

File Mode  ([F]ile)                                 *vproj-mode-file*
    Browse the filesystem. Directories listed first, then files.
    File sizes shown for regular files. Press |l| or Enter on a
    directory to navigate into it, on a file to open it. Press |h|
    or . to go to the parent directory.

    Press |T| to toggle tree view (indented with expand/collapse).
    Press |p| to toggle a file preview split that updates on cursor move.
    Press |/| to filter by name pattern. Press |*| to grep the project
    and populate quickfix. Press |Ctrl-G| to toggle showing only
    git-changed files.

                                               *vproj-tree-view*
    Tree view toggles between flat listing and an indented directory
    tree with expand/collapse within file mode.

                                               *vproj-preview*
    File preview shows file contents in a split that updates as the
    cursor moves. Press |p| again to close.

                                               *vproj-filter*
                                               *vproj-grep*
    Filter (|/|) narrows the listing by name pattern. Grep (|*|) runs
    `grep -rn` and populates the quickfix list for |vproj-mode-qfix|.

Buf Mode  ([B]uf)                                   *vproj-mode-buf*
    Switch between open buffers. Modified buffers marked with +,
    current buffer marked with %. Press |x| to close a buffer.
    Line counts shown.

Git Mode  ([G]it)                                   *vproj-mode-git*
    Project-aware file tree from a .vproj file. Included items
    shown normally; excluded items shown in parentheses.
    Press |+| or |-| to toggle inclusion. Press Enter on the
    status line to rename or create the project.

    Git actions: |s| to stage/unstage, |d| for diff preview,
    |D| to discard changes, |C| to commit, |P| to push,
    |U| to pull --ff-only, |B| to switch branches.

                                               *vproj-git-actions*
    Git actions work in file, git, and log modes. Stage (|s|)
    uses `git add` / `git reset HEAD`. Diff preview (|d|) opens
    a vertical split with `git diff`. Discard (|D|) prompts for
    confirmation before `git checkout`. Commit (|C|) prompts for
    a message. Push (|P|), pull (|U|), and branch switch (|B|)
    use prompts for target/confirmation.

                                               *vproj-git-stash*
    Stash (|z|) pushes current changes with an optional message via
    `git stash push`. Stash pop (|Z|) lists all stashes first, then
    prompts for which to pop (default: top). Both operate on the
    whole repository.

                                               *vproj-git-blame*
    Blame (|a|) opens a vertical split showing `git annotate` output
    for the file under cursor. Requires the file be tracked by git.
    Press |q| in the blame window to close it. Only in file mode.

Qfix Mode  ([Q]fix)                                 *vproj-mode-qfix*
    Displays the Vim quickfix list. Press Enter to open the
    selected entry at the file and line number.

Log Mode  ([L]og)                                   *vproj-mode-log*
    Git commit history. Shows `git log --oneline` output with
    nav characters. Press Enter on a commit to open a detailed
    diff in a split. Press |L| to switch to Log mode.

Press Enter on the mode menu line (line 1) to cycle through modes.
Press |f|, |b|, |g|, |q|, or |L| inside the pane to switch directly.

4. KEY MAPPINGS                                    *vproj-mappings*

All mappings are buffer-local to the VPROJ pane. They do not affect
other buffers.

Navigation:
    |j| / |k|           Move selection down / up
    <Down> / <Up>       Same as j / k
    |h|                 Go to parent directory
    |l|                 Open file or enter directory
    Enter               Same as l
    |.|                  Go to parent directory (same as h)
    <C-T>               Jump to first item
    <C-B>               Jump to last item
    <C-K>               Go to parent directory (same as h)
    <C-J>               Enter first subdirectory

Mode switching:
    |f|                 File mode
    |b|                 Buf mode
    |g|                 Git mode
    |q|                 Qfix mode
    |L|                 Log mode

Git actions (file and log modes):
    |s|                 Stage / unstage file under cursor
    |d|                 Open diff preview in vertical split
    |D|                 Discard file changes (with confirmation)
    |C|                 Commit with message prompt
    |P|                 Push to remote
    |U|                 Pull --ff-only from remote
    |B|                 Switch branch (with prompt)
    |z|                 Stash changes (optional message)
    |Z|                 Pop a stash (shows list, select by index)
    |a|                 Blame file under cursor (git annotate)
    <C-G>               Toggle showing only git-changed files

Actions:
    |r|                 Refresh listing
    |x|                 Close selected buffer (buf mode)
    |+| / |-|           Include / exclude item (git mode)
    |T|                 Toggle tree view (file mode)
    |p|                 Toggle file preview split
    |/|                 Filter files by name pattern
    |*|                 Grep project and populate quickfix
    <Left> / <Right>    Shrink / grow pane width
    <F1>                Toggle info column

Quick Nav:                                       *vproj-nav-char*
    >                   Shift nav indicators forward
    <                   Shift nav indicators backward
    |a|-|z|, |A|-|Z|, |1|-|9|  Jump to item by nav character

                                               *vproj-paging*
Paging:
    <C-N>               Next page
    <C-P>               Previous page

Close:
    |Q|                 Close pane
    |F4|                Close pane
    |Esc|               Close pane (temporary mode only)

Global (outside pane):
    <Tab>               Open pane (temporary mode, auto-closes)
    <S-Tab>             Open pane (permanent mode, stays open)

Pane Modes:                                      *vproj-pane-modes*
    Temporary mode (Tab): The pane auto-closes when you open a file,
    press |Esc| in the pane, or press |Tab| again. This mode is for
    quick file/buffer access — no extra keystroke to dismiss.

    Permanent mode (Shift-Tab): The pane stays open until you press
    |Shift-Tab| again, press |q| (with the pane focused), or close
    the pane buffer via |:bdelete|. This mode is for extended project
    management sessions.

    Pressing the alternate toggle key while the pane is open
    transitions between modes without closing the pane.

5. CONFIGURATION                                   *vproj-config*

                                               *g:vproj_show_dotfiles*
    g:vproj_show_dotfiles
        Set to 1 to show dotfiles (hidden files starting with ".").
        Default: hidden.

                                               *g:vproj_pane_width_default*
    g:vproj_pane_width_default
        Initial pane width in columns (20-80). Default: 40.

                                               *g:vproj_pane_width_file*
    g:vproj_pane_width_file
        Pane width for File mode (20-80). 0 = use default. Default: 0.

                                               *g:vproj_pane_width_buf*
    g:vproj_pane_width_buf
        Pane width for Buf mode (20-80). 0 = use default. Default: 0.

                                               *g:vproj_pane_width_git*
    g:vproj_pane_width_git
        Pane width for Git mode (20-80). 0 = use default. Default: 0.

                                               *g:vproj_pane_width_qfix*
    g:vproj_pane_width_qfix
        Pane width for Qfix mode (20-80). 0 = use default. Default: 0.

                                               *g:vproj_pane_width_log*
    g:vproj_pane_width_log
        Pane width for Log mode (20-80). 0 = use default. Default: 0.

                                               *vproj-highlights*
Highlight groups:
    VprojModeFile       File mode label (yellow, bold, underline)
    VprojModeBuf        Buf mode label (green, bold, underline)
    VprojModeGit        Git mode label (magenta, bold, underline)
    VprojModeQfix       Qfix mode label (blue, bold, underline)
    VprojModeLog        Log mode label (cyan, bold, underline)
    VprojCursorLine     Selected line highlight (reverse video)
    VprojNavIndicator   Nav character indicator (cyan)
    VprojInfoColumn     File size / buffer info column (green)
    VprojParentDir      Parent directory entry (blue)
    VprojDirName        Directory name (bold)
    VprojSeparator      Separator line (dark grey)
    VprojStatusLine     Git mode status line (reverse video)

                                               *vproj-troubleshooting*
Troubleshooting ~

                                               *vproj-cmdheight*
    E36 "Not enough room" on split ~
        If you see E36 "Not enough room" when opening files, blame,
        or diffs from the pane, check your command-line height:
>
            :set cmdheight?
<
        If the value is unusually high (above 2), Vim may not have
        enough window space to create vertical splits. Lower it:
>
            :set cmdheight=1
<
        Or set it permanently in your vimrc:
>
            set cmdheight=1
<
        Some terminal emulators and multiplexers configure high
        cmdheight values automatically. vproj temporarily lowers
        cmdheight, winminwidth, and winminheight during split
        operations to prevent E36, but persistent high values can
        still cause issues in edge cases.

        If the problem persists, run |:VprojDiag| to see current
        window dimensions and option values.

Example vimrc:
>
    " Show hidden files
    let g:vproj_show_dotfiles = 1

    " Remap temporary toggle to F2
    nmap <F2> <Plug>VprojToggle
    nunmap <Tab>

    " Remap permanent toggle to F3
    nmap <F3> <Plug>VprojTogglePermanent
    nunmap <S-Tab>
<

6. .vproj FILE FORMAT                             *vproj-file-format*

The .vproj file defines a project for Git Mode. It lives at the
project root.

Format: >

    Project Name: my-project
    Project Root: /home/user/dev/my-project
    Included Directories:
    src
    Included Files:
    README.md
    Excluded Directories:
    .git
    node_modules
    Excluded Files:
    .env

Lines starting with # are comments.
