# VPN Phase 2+3 — File, Git, and Symbols Modes

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development

**Goal:** Add File mode (project root detection + flat labeled file tree with paging), Git mode (status categories with stage/unstage/diff), and Symbols mode (LSP document symbols with label navigation).

**Architecture:** Three independent mode+adapter pairs. Each mode implements the Mode interface from Phase 1. Adapters isolate external system dependencies (git CLI, LSP, tree-sitter). All modes register into the existing mode registry. Integration into init.lua is done after all modules land.

---

## Agent 1: Git Adapter + Git Mode

### Files:
- Create: `lua/vpn/adapters/git.lua`
- Create: `lua/vpn/modes/git.lua`
- Create: `tests/spec/git_mode_spec.lua`

### git.lua — Adapter
- `get_status(cwd)` — runs `git -C <cwd> status --porcelain`, returns `{staged={}, unstaged={}, untracked={}, conflicts={}}` keyed by filename+status
- `stage(file)` — `git add <file>`
- `unstage(file)` — `git reset HEAD <file>`
- `diff(file)` — returns `git diff` output for file
- `commit(msg)` — `git commit -m <msg>`
- Graceful degradation when not in a git repo

### git.lua — Mode
- Mode interface: {name="Git", key="g", icon="G", enabled=true}
- refresh() calls adapter.get_status()
- render() generates labeled lines with status prefixes: `M`, `A`, `??`, `UU`
- select(label) jumps to file
- Actions: `s`=stage, `u`=unstage, `d`=diff preview, `c`=commit

## Agent 2: Project Root + File Mode

### Files:
- Create: `lua/vpn/core/project.lua`
- Create: `lua/vpn/modes/files.lua`
- Create: `tests/spec/files_mode_spec.lua`

### project.lua — Project Root Detection
- `find_root()` — walks up from cwd looking for markers: .git, package.json, pyproject.toml, Cargo.toml, Makefile, etc.
- `scan_files(root)` — returns flat list of project files (respects .gitignore), cached
- `get_relative_path(file)` — file path relative to project root

### files.lua — Mode
- Mode interface: {name="Files", key="f", icon="F", enabled=true}
- refresh() calls project.find_root() + project.scan_files()
- render() generates labeled flat file list with paging
- select(label) opens file
- Paging: `[`/`]` for prev/next page, page indicator in header
- Directories shown with trailing `/`

## Agent 3: LSP + Tree-sitter Adapters + Symbols Mode

### Files:
- Create: `lua/vpn/adapters/lsp.lua`
- Create: `lua/vpn/adapters/treesitter.lua`
- Create: `lua/vpn/modes/symbols.lua`
- Create: `tests/spec/symbols_mode_spec.lua`

### lsp.lua — LSP Adapter
- `get_document_symbols(buf)` — calls `vim.lsp.buf_request` for `textDocument/documentSymbol`
- `get_workspace_symbols(query)` — calls `vim.lsp.buf_request` for `workspace/symbol`
- Returns normalized symbol list: {name, kind, line, col, uri}
- Graceful fallback when no LSP attached

### treesitter.lua — Tree-sitter Fallback
- `get_symbols(buf)` — uses tree-sitter to extract functions, classes, methods
- Returns same normalized format as LSP adapter
- Graceful fallback when tree-sitter not available

### symbols.lua — Mode
- Mode interface: {name="Symbols", key="s", icon="S", enabled=true}
- refresh() tries LSP first, falls back to tree-sitter
- render() generates labeled symbol list with kind icons
- select(label) jumps to symbol definition
- Symbol kind indicators: `[C]` class, `[F]` function, `[M]` method, `[V]` variable
