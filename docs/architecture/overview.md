# Architecture Overview

Vproj is a pure Vim9Script plugin following a modular, event-driven architecture.

## Module Map

```
autoload/vproj/
├── init.vim              Setup(), Toggle(), Open(), Close()
├── config.vim            Defaults and deep merge
├── sidebar.vim           Window and buffer management
├── renderer.vim          Header, separator, body rendering
├── labels.vim            DSN label generation engine
├── navigation.vim        Key dispatch and mapping
├── handler.vim           Legacy bridge for mappings
├── project.vim           Root detection and file scanning
├── persistence.vim       Session save/restore (JSON)
├── workspace.vim         Pins, bookmarks, named workspaces
├── modes.vim             Mode registry
├── buffers_mode.vim      Buffer list mode
├── files_mode.vim        File tree mode
├── git_mode.vim          Git status mode
├── git.vim               Git porcelain parser
├── symbols_mode.vim      Ctags symbol mode
├── outline_mode.vim      Filetype-aware outline
├── events.vim            Event bus
└── cache.vim             TTL cache

plugin/
└── vproj.vim             Entry point and commands
```

## Design Principles

### Renderer Owns All Screen Updates

No mode performs direct rendering. The renderer (`renderer.vim`) owns all
screen updates. Modes produce data (lines and label maps); the renderer
writes them to the sidebar buffer.

### Event Bus Decouples Subsystems

The event bus (`events.vim`) decouples subsystems. Modes emit events;
listeners in `init.vim` react. Modes don't know about the sidebar, the
renderer, or the navigation engine.

### Labels Are Addresses, Not Ordering

The label map is a dict (`label -> item`), so selection is O(1). Labels
are generated by the label engine (`labels.vim`) and assigned to items
by position, not by content.

### TTL-Based Caching

Project file trees, git status, and ctags data are cached with configurable
TTLs (`cache.vim`). Cache entries expire automatically; no manual
invalidation needed.

### Mode Interface

Every mode implements a standard dict interface:

```
Mode = {
    name:    string        # Display name (e.g., 'Git')
    key:     string        # Single-char hotkey (e.g., 'g')
    icon:    string        # Single-char display icon
    enabled: bool          # Whether this mode is active
    Refresh: func()        # Gather current data
    Render:  func(): dict  # Produce {lines, label_map}
    Select:  func(label): any  # Handle label selection
}
```

## Data Flow

1. **User presses hotkey** → `plugin/vproj.vim` calls `vproj#init#Toggle()`
2. **Toggle opens sidebar** → `sidebar.vim` creates a scratch buffer
3. **Default mode loads** → `modes.vim#Switch()` fires `mode_changed` event
4. **Mode renders** → mode's `Refresh()` gathers data, `Render()` produces lines
5. **Renderer draws** → `renderer.vim#RenderFull()` writes to sidebar buffer
6. **Navigation attaches** → `navigation.vim#Attach()` maps label keys
7. **User presses label** → `handler.vim` bridge → `navigation.vim#Dispatch()` → mode's `Select()`
